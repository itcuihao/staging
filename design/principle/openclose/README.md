# 开闭原则

>软件实体应该对扩展开放，对修改关闭。一个软件实体应该通过扩展来实现变化，而不是通过修改已有的代码来实现变化。

## 多扩展开放，修改关闭

- 逻辑变化
- 子模块变化
- 可见视图变化

## 为什么用开闭原则

- 开闭原则对测试的影响

保证不产生有毒代码（Poisonous Code）

- 开闭原则可以提高复用性

- 开闭原则可以提高可维护性

- 面向对象开发的要求

## 如何使用开闭原则

1.抽象约束

- 不出现接口不存在的public func
- 尽量使用接口，go中虽然没有继承，但是可以用组合
- 抽象层尽量稳定

2.元数据控制模块行为
3.制定项目章程
4.封装变化
